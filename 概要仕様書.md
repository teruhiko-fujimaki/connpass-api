

# 概要仕様書（改訂版Ver.2）

## 1. システム概要

本システムは、IT勉強会プラットフォーム「connpass」で**現在募集中のイベントデータ**をリアルタイムに取得・集計し、Google マップ上で市・区単位の開催密度をグラフィカルに可視化するWebアプリケーションである。
過去のデータや期間指定などの複雑な操作を排除し、「今、どの地域でイベントが多く募集されているか」を直感的に把握できるシンプルなUI/UXを提供する。

## 2. 外部API連携・セキュリティ仕様

### ① Google Maps API

* **用途:** メイン画面への地図描画、およびエリアごとの開催数に応じたバブル（円マーカー）のレンダリング。
* **認証・セキュリティ:**
* 取得したGoogle Maps APIキーは、プロジェクトルートの環境変数設定ファイル（`.env`）に埋め込んで管理する。
* 本番環境へのデプロイ時は、Google Cloud Console側で「HTTPリファラ（使用するドメイン）」によるAPIキーの利用制限を必ず設定する。



### ② connpass API（v1 / v2）

提供元仕様（ [https://connpass.com/about/api/v2/](https://connpass.com/about/api/v2/) ）に基づく連携仕様。

* **仕様概要:**
* 本APIは認証用のAPIキーが不要であり、エンドポイントに対して直接リクエスト（GET形式）を送信することで、イベント情報をJSON形式で取得できる公開APIである。


* **データ取得・フィルタリング:**
* **「現在募集中」のイベントのみ**に対象を絞るため、APIリクエスト時に、現時点（または本日以降）の日時パラメーターや、募集ステータスに関する条件を組み合わせて現在進行形のイベントのみを抽出する（過去の開催イベントは取得・表示の対象外とする）。


* **利用データ範囲:**
* イベントの開催場所（`address`）から「市」「区」の住所文字列を抽出。
* 各自治体ごとの「募集中のイベント総数」のみを集計（※イベントのジャンル内訳、参加者数、平日/土日の区別は使用しない）。



## 3. 主要機能要件

### ① データ集計・管理機能

* **リアルタイム地域集計:** 取得した「現在募集中」のイベントデータを住所文字列ベースで各自治体の中心座標に紐付け、市・区ごとの合計件数を動的にカウントする。

### ② マップ可視化機能（バブルチャート表示）

* **カスタムバブルマーカー:** 各市・区の中心点に、そのエリア内の「募集中イベント数」に応じた大きさのシンプルな正円（バブル）を表示する。
* **ビジュアル表現:** イベント数が多い地域ほど円の半径が大きくなり、一目でイベントの集中度がわかるようにする。円には適度な透過度（不透明度）を設定し、バブル同士が重なっても背後の地図や隣のエリアが見えるよう配慮する。

## 4. 画面レイアウト構成（UI仕様）

複雑な操作パネル（スライダーやトグル）が不要になったため、画面全体をGoogle マップが占める、極めてシンプルなフルスクリーンレイアウト（またはそれに準ずる形）とします。

```
+--------------------------------------------------------------------------+
|  アプリケーションタイトル (connpass 募集中イベントリアルタイムマップ)    |
+--------------------------------------------------------------------------+
|                                                                          |
|                                                                          |
|                                                                          |
|                                                                          |
|                        メインエリア（画面のほぼ全体）                     |
|                                                                          |
|                                                                          |
|                              Google マップ                               |
|                                                                          |
|          （各市・区の中心に、募集数に応じた半透明のバブルを表示）        |
|                                                                          |
|                                                                          |
|                                                                          |
|                                                       [凡例]             |
|                                                       ○ 小（1〜2件）      |
|                                                       ◯ 中（5〜10件）     |
|                                                       ◯ 大（20件〜）      |
+--------------------------------------------------------------------------+

```

* **メインエリア:**
* 画面のほぼ全体（10割）にGoogle マップを配置。
* ユーザーは地図を拡大・縮小・移動しながら、今盛り上がっている地域を探索する。


* **周辺UI:**
* 画面の隅に、バブルの大きさが何件程度を示しているかを表すシンプルな「凡例」のみを小さく配置。

## 5. 技術選定

* **フロントエンド:** HTML5, CSS3, JavaScript / TypeScript
* **地図API:** Google Maps JavaScript API (環境変数からキーを読み込んでロード)
* **バブル描画:**
* Google Maps標準の `google.maps.Circle` を使用（これにより、地図のズームレベルに関わらず一貫した表現や、軽量な描画が容易になります）。
* または最新の `AdvancedMarkerElement` を使い、CSSで丸（`border-radius: 50%`）のサイズをイベント数に応じて可視化する。


* **環境変数管理:** `.env` ファイルを使用。
